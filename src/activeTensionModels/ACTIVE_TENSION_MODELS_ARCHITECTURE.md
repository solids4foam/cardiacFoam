# Active Tension Models — Electro-Mechanical Coupling Architecture

## 1. Overview

The active tension model layer computes **mechanical stress generated by cardiac muscle contraction** in response to electrical activation. It translates from electrical state (voltage, intracellular calcium) to mechanical output (active stress/strain), enabling electro-mechanical coupling simulations.

This directory provides:
- **activeTensionModel base class** — abstract interface for contraction models
- **Concrete implementations** — phenomenological (GoktepeKuhl, NashPanfilov) and cellular-level models
- **Runtime selection** — switch between models without recompilation
- **Integration with electroModel** — bidirectional coupling via electroMechanicalModel wrapper

## 2. activeTensionModel Base Class Interface

### Core Virtual Methods

```cpp
class activeTensionModel
{
public:
    // Compute active Cauchy stress given electrical state
    virtual tmp<volSymmTensorField> sigma_active
    (
        const volScalarField& Vm,        // Transmembrane voltage
        const volScalarField& Cai,       // Intracellular calcium [M]
        const volScalarField& fiber_stretch  // Muscle fiber stretch λ
    ) const = 0;

    // Alternative: return active strain (material description)
    virtual tmp<volSymmTensorField> epsilon_active
    (
        const volScalarField& Vm,
        const volScalarField& Cai,
        const volScalarField& fiber_stretch
    ) const;

    // Advance internal state (e.g., cross-bridge dynamics)
    virtual void advanceActivation
    (
        scalar dt,
        const volScalarField& Vm,
        const volScalarField& Cai,
        volScalarField& lambda_xb  // Cross-bridge fraction (if model uses it)
    );

    // Return model info for output
    virtual word type() const = 0;
    virtual wordList exportVariables() const;
};
```

### Input Variables

| Variable | Units | Source | Meaning |
|----------|-------|--------|---------|
| **Vm** | V | Ionic model / myocardium | Transmembrane voltage |
| **Cai** | M (molar) | Ionic model | Intracellular calcium concentration |
| **fiber_stretch** | dimensionless | Solid mechanics solver | Current muscle fiber stretch λ = current length / rest length |

### Output Variables

| Variable | Units | Destination | Meaning |
|----------|-------|-------------|---------|
| **sigma_active** | Pa (N/m²) | Solid mechanics (stress) | Cauchy stress in stretched (deformed) configuration |
| **epsilon_active** | dimensionless | Solid mechanics (strain) | Green-Lagrange strain in reference config |
| **lambda_xb** | [0,1] | Internal state | Cross-bridge activation fraction (driving force for contraction) |

## 3. Concrete Model Implementations

### A. GoktepeKuhl Model (Phenomenological)

**Reference:** Göktepe & Kuhl (2010) — energy-based approach

**Concept:**
- Active stress is a **function of calcium and stretch** only
- No explicit time-dependence; instantaneous response
- Single scalar internal variable: cross-bridge activation λ_xb ∈ [0,1]

**Formulation:**

```
σ_active = σ_0 × λ_xb(Cai) × f(λ_fiber)

where:
  σ_0 = reference maximum active stress (~30 kPa)
  λ_xb(Cai) = calcium-dependent activation (S-shaped curve)
  f(λ) = fiber-stretch modulation (peaks near stretch ~1.2)
```

**Calcium response:**

```
λ_xb(Cai) = [Cai]⁴ / ([Cai_50]⁴ + [Cai]⁴)

where [Cai_50] ≈ 0.5 μM (typical heart)
```

**Stretch dependence:**

```
f(λ) = exp(-k × (λ − λ_optimal)²)

where:
  λ_optimal ≈ 1.2 (maximum force near ~20% stretch)
  k ≈ 15 (width of force-stretch relation)
```

**Advantages:**
- ✅ Simple (2-3 parameters)
- ✅ Efficient (no internal ODEs to advance)
- ✅ Matches whole-heart experiments reasonably
- ⚠️ Lacks cross-bridge dynamics (calcium-induced force rise time)

### B. NashPanfilov Model (Phenomenological)

**Reference:** Nash & Panfilov (2004) — simplified cellular model

**Concept:**
- Active stress with explicit **time-dependence** via internal state variable
- Cross-bridge kinetics model tension development after Ca2+ rise
- Enables force-length-velocity relations

**State Variable:**

```
dλ_xb/dt = (λ_xb_∞(Cai) − λ_xb) / τ_xb(Cai, λ_fiber)

where:
  λ_xb_∞ = equilibrium activation level (as in Göktepe)
  τ_xb = time-constant (depends on Ca and stretch)
```

**Advantages:**
- ✅ Includes temporal dynamics (contraction kinetics)
- ✅ Can capture force-frequency relation
- ✅ Still computationally cheap (single ODE per cell)
- ⚠️ More parameters to fit

### C. Cellular-Level Models (Future)

Proposed but not yet implemented:
- **Rice model:** Explicit cross-bridge binding kinetics (e.g., A.M.xp, A.M, D.M states)
- **Campbell model:** Full myosin power-stroke dynamics
- **Hunter model:** Coupled with intracellular calcium handling

These would require:
- 10+ internal state variables per cell
- Coupling to calcium from ionic model
- Higher computational cost

## 4. Mechanical Stress Formulation

### Passive (Elastic) Stress

From solid mechanics (strain energy function):

```
σ_passive = ∂W_passive / ∂E

where W_passive = W_iso(I₁, I₂) + W_vol(J)
      I₁, I₂ = invariants of deformation tensor
      J = volume change
```

Examples: Mooney-Rivlin, Holzapfel-Ogden (fiber-oriented)

### Active (Contractile) Stress

From activation:

```
σ_active = σ_0(Vm, Cai) × f_len(λ_fiber, λ_sheet) × (n_fiber ⊗ n_fiber)

where:
  n_fiber = fiber direction (material coordinate)
  σ_0 = peak active stress
  f_len = length-dependent modulation
```

**Key point:** Active stress acts **only along fiber direction** (nearly 1D contraction).

### Total Solid Mechanics Equation

```
ρ₀ ∂²u/∂t² = ∇·(σ_passive + σ_active) + f_body

where u = displacement field
```

Solved by mechanics solver (solids4foam), not electroModels.

## 5. Integration with electroMechanicalModel

### Wrapper Class

The **electroMechanicalModel** owns both:

```cpp
class electroMechanicalModel : public physicsModel
{
    autoPtr<electroModel> electroModel_;           // Electrical domain
    autoPtr<activeTensionModel> activeTensionModel_;  // Mechanical model
    autoPtr<mechanicalModel> mechanicalModel_;     // Solid mechanics (external)
};
```

### Coupling Loop

At each time step:

```
1. Ionic step:      Solve ionic ODE → get Vm(n+1), Cai(n+1)
2. Electrical PDE:  Solve monodomain/bidomain → updated Vm(n+1)
3. Calcium → Tension: activeTensionModel(Vm, Cai) → σ_active
4. Mechanical PDE:  Solid mechanics solve with σ_active as RHS → displacement u(n+1)
5. Update geometry: λ_fiber(n+1) from computed deformation
6. Next step
```

### Configuration: electroProperties

```cpp
physicsModel  electroMechanicalModel;

electroMechanicalModelCoeffs
{
    // Electrical settings (all monodomainSolverCoeffs options)
    myocardiumSolver    monodomainSolver;
    monodomainSolverCoeffs { ... }

    // Mechanical settings
    activeTensionModel  GoktepeKuhl;
    activeTensionCoeffs
    {
        sigma_max     [1 -1 -2 0 0 0 0]  30000;   // [Pa] peak active stress
        lambda_opt    1.2;                        // Optimal stretch
        Ca_50         1e-6;                       // [M] calcium sensitivity
    }
}
```

## 6. Parameter Fitting and Validation

### Experimental Data Sources

| Measurement | Protocol | Model Target |
|-------------|----------|--------------|
| **Force-Calcium** | Isolated myocyte at fixed length | σ_active(Cai) curve |
| **Force-Length** | Whole muscle at varying stretch | f_len(λ) at rest Ca |
| **Force-Frequency** | Paced muscle at different rates | τ_xb (kinetic time constant) |
| **Twitch kinetics** | Step Ca (flash photolysis) → force rise | Time-constants (if time-dependent model) |

### Fitting Workflow

1. **Extract cell-level data:** Single cell/fiber at fixed conditions
2. **Run singleCellSolver** with same Ca protocol
3. **Adjust σ_max, Ca_50, λ_opt** to match force vs. Ca
4. **Run single-beat simulation** with twitch stimulus
5. **Compare force rise time** against experiment
6. **Iterate** parameters

### Typical Parameters

| Parameter | Units | Göktepe | Nash | Notes |
|-----------|-------|---------|------|-------|
| σ_max | kPa | 30 | 50 | Species, cell type dependent |
| Ca_50 | μM | 0.5–1.0 | 0.7 | Troponin sensitivity |
| λ_opt | — | 1.15–1.25 | 1.2 | Optimal sarcomere length |
| τ_ca | ms | — | 10–20 | Ca-driven kinetics (Nash only) |

## 7. Challenges and Limitations

### Challenge 1: Multi-Scale Representation

**Cross-bridge level:** Individual myosin motor transitions (μs timescale, nm length scale)

**Sarcomere level:** Collective cross-bridge behavior → net force (ms timescale, μm length scale)

**Cell level:** Myofibril network → whole-cell force (ms, mm)

**Tissue level:** Fiber sheets, arrangement → organ contraction (s, cm)

Phenomenological models (Göktepe, Nash) **bridge scales** by fitting parameters, but lose mechanistic detail.

### Challenge 2: Anisotropy

Heart muscle is **highly anisotropic:**
- Fiber direction (fast conduction, strong contraction)
- Sheet direction (secondary structure)
- Cross-fiber direction (weakest)

Most active tension models act **only in fiber direction**, but:
- ⚠️ Ignores sheet mechanics
- ⚠️ Can over-stiffen perpendicular directions

### Challenge 3: Calcium-Force Temporal Mismatch

- **Electrical:** Vm changes in ~100 ms (action potential)
- **Calcium:** Rises/falls in ~200–500 ms (SR release/reuptake)
- **Force:** Rises in ~100–150 ms, falls in 300–600 ms (cross-bridge kinetics)

Phenomenological models cannot fully capture this mismatch; cellular models needed.

## 8. Extension Points

### Adding a New Active Tension Model

1. Inherit `activeTensionModel`
2. Implement `sigma_active()` (or `epsilon_active()`)
3. If time-dependent: implement `advanceActivation()`
4. Define parameters and their units
5. Register: `addToRunTimeSelectionTable(activeTensionModel, MyModel, dictionary)`
6. Test against literature or experimental data

### Example: Custom Model

```cpp
class MyActiveTension : public activeTensionModel
{
    scalar sigma_max_;
    scalar Ca_50_;
    scalar lambda_opt_;

public:
    MyActiveTension(const dictionary& dict);

    virtual tmp<volSymmTensorField> sigma_active
    (
        const volScalarField& Vm,
        const volScalarField& Cai,
        const volScalarField& fiber_stretch
    ) const override;
};
```

## 9. Connection to Solid Mechanics (solids4foam)

The **electroMechanicalModel** interfaces with the **mechanics solver** (solids4foam):

### Handoff Points

1. **Active stress inlet:** electroModel → σ_active → mechanics solver
2. **Geometry update:** mechanics solver solves u → sends back current fiber stretch λ
3. **Feedback loop:** λ_fiber(n+1) → next electrical step uses updated geometry

### Solid Mechanics Solver (External)

Solves:

```
ρ₀ ∂²u/∂t² = ∇·(σ_passive + σ_active) + f_body

Boundary conditions: u or traction specified
Geometry update: x_new = x_old + u
```

Not part of electroModels; handled by solids4foam's finite-strain solid mechanics.

## 10. Key Design Decisions

### Why Separate Active Tension Model?

- Decouples electrophysiology from mechanics
- Users can run electrical-only (faster) or electro-mechanical
- Different computational costs: mechanics ≫ electrical
- Easy to swap tension models without affecting electrical code

### Why Not Include Mechanics in electroModel?

- Solid mechanics (FEM, finite strain, geometry tracking) is **very different** from PDE solving
- solids4foam is mature, specialized; reusing it is better than reimplementing
- electroModel is small, focused; mechanics is large, separate concern

### Why Phenomenological Over Cellular?

- Cellular models (Rice, Campbell) have **10+ ODEs** per cell → 10× cost
- Phenomenological models (Göktepe, Nash) capture **qualitative behavior** well
- For whole-heart simulations, phenomenological is **good enough** and much faster
- Cellular models valuable for **specific research** (single-cell mechanics, drug response)

## 11. Summary

Active tension models bridge **electrical activation → mechanical contraction**:

| Component | Purpose | Example |
|-----------|---------|---------|
| **Base class** | Common interface | `activeTensionModel::sigma_active()` |
| **Phenomenological** | Efficient, calibrated | Göktepe (instantaneous), Nash (kinetic) |
| **Parameters** | Tissue-specific values | σ_max, Ca_50, λ_opt |
| **Integration** | Time-stepping with ionic | `advanceActivation()` for internal states |
| **Coupling** | To solid mechanics | σ_active → fiber stretch feedback |

They enable **multi-physics simulation** of heart function: from ion channels to whole-organ mechanics.
